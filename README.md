## 项目简介
本项目是《软件工程课程实践》实验 5 的实现程序，基于 PERT（Program Evaluation and Review Technique，计划评审技术）对项目任务网络进行分析。程序能够根据用户输入的任务信息和前置依赖关系，自动完成任务期望工期计算、关键路径分析、项目总工期分析、任务总时差计算，并在保证项目最短工期不变的前提下，计算完成项目所需的最少人数并生成任务分配安排表。

本系统采用原生 `HTML + CSS + JavaScript` 开发，具有界面直观、操作简单等特点。

## 主要功能
本程序实现了以下功能：

- 支持输入任务 ID、任务名称、三点估计时间 `O/M/P`
- 支持输入任务之间的前置依赖关系
- 自动计算任务期望工期 `TE`
- 自动计算最早开始时间 `ES`、最早完成时间 `EF`
- 自动计算最晚开始时间 `LS`、最晚完成时间 `LF`
- 自动计算任务总时差 `TF`
- 自动识别关键任务
- 自动提取关键路径
- 自动计算项目总工期
- 自动搜索满足最短工期条件下的最少人数
- 自动生成人员任务分配安排表
- 以甘特图式时间视图展示排班结果
- 支持任务数据的 JSON 导入与导出
- 支持多个起始任务和多个结束任务

## 输入说明

程序采用表格形式录入任务数据。每个任务需要填写以下字段：

- `Task ID`：任务编号，必须唯一
- `任务名称`：任务名称或任务描述
- `O 乐观`：任务在最理想情况下所需时间
- `M 最可能`：任务在正常情况下最可能所需时间
- `P 悲观`：任务在最不利情况下所需时间
- `前置任务`：该任务开始前必须完成的任务，填写任务 ID，多个前置任务之间使用英文逗号分隔

例如：

- `A | 需求分析 | 2 | 4 | 6 |`
- `B | 数据库设计 | 3 | 5 | 9 | A`

## 程序工作原理

### 1. 输入阶段（Input）

用户输入任务的基本信息和任务之间的前置依赖关系。程序支持手动录入，也支持通过 JSON 文件导入已有任务数据。

输入内容包括：

- 任务 ID
- 任务名称
- 乐观时间 `O`
- 最可能时间 `M`
- 悲观时间 `P`
- 前置任务列表

### 2. 处理阶段（Processing）

#### 2.1 数据合法性校验

程序首先对输入数据进行校验，主要检查以下内容：

- 任务 ID 是否为空
- 任务 ID 是否重复
- 任务名称是否为空
- `O`、`M`、`P` 是否为非负数字
- 是否满足 `O <= M <= P`
- 前置任务是否真实存在
- 依赖关系中是否存在循环依赖

如果校验失败，程序会提示错误信息并停止后续计算。

#### 2.2 期望工期计算

对于每个任务，程序根据 PERT 的三点估计公式计算期望工期：

```text
TE = (O + 4M + P) / 6
```

其中：

- `O` 表示乐观时间
- `M` 表示最可能时间
- `P` 表示悲观时间
- `TE` 表示任务期望工期

#### 2.3 前向计算

程序将所有任务及其依赖关系构建成任务网络图，并利用拓扑排序进行前向计算，求出：

- `ES`：最早开始时间
- `EF`：最早完成时间

计算规则如下：

- 无前置任务的任务：`ES = 0`
- 有前置任务的任务：`ES = max(所有前置任务的 EF)`
- `EF = ES + TE`

#### 2.4 后向计算

在得到项目总工期后，程序对任务网络执行后向计算，求出：

- `LS`：最晚开始时间
- `LF`：最晚完成时间

计算规则如下：

- 对于结束任务：`LF = 项目总工期`
- 对于其他任务：`LF = min(所有后继任务的 LS)`
- `LS = LF - TE`

#### 2.5 时差与关键路径分析

在得到最早时间和最晚时间后，程序计算任务总时差：

```text
TF = LS - ES
```

判断规则：

- 若 `TF = 0`，则任务为关键任务
- 连续的关键任务链构成关键路径

项目总工期等于所有结束任务最早完成时间中的最大值。

#### 2.6 最少人数与任务排班

程序假设：

- 每个任务固定需要 1 人执行
- 任务开始后不可中断
- 同一时刻同一人不能执行多个任务
- 项目总工期不能延长

在此前提下，程序会在每个任务允许的开始区间内进行可行性搜索，从 `1` 人开始逐步尝试，直到找到满足最短工期要求的最少人数，并生成每个人员在不同时间段的任务安排。

### 3. 输出阶段（Output）

程序最终输出以下结果：

- 每个任务的期望工期 `TE`
- 每个任务的 `ES / EF / LS / LF / TF`
- 关键任务列表
- 关键路径
- 项目总工期
- 最少人数
- 人员任务分配安排表
- 甘特图式时间视图

## 界面与输出说明

### 1. 任务输入区

用户通过表格输入任务信息和前置任务关系。

### 2. 项目摘要区

显示：

- 项目总工期
- 最少人数
- 关键任务数量
- 任务总数

### 3. 任务结果表

显示每个任务的：

- ID
- 名称
- O / M / P
- TE
- 前置任务

- LS / LF
- TF
- 是否为关键任务

### 4. 任务分配安排表

按人员显示任务安排，包括：

- 人员编号
- 任务 ID
- 任务名称
- 开始时间
- 完成时间
- 是否关键任务

### 5. 甘特图式时间视图

系统按人员分组展示任务时间轴，并对关键任务使用不同颜色进行高亮，便于直观观察项目安排和并行情况。

## JSON 导入导出格式

程序支持将任务数据保存为 JSON 文件，并支持重新导入。

示例格式如下：

```json
{
  "tasks": [
    {
      "id": "A",
      "name": "需求分析",
      "optimistic": 2,
      "mostLikely": 4,
      "pessimistic": 6,
      "predecessors": []
    },
    {
      "id": "B",
      "name": "数据库设计",
      "optimistic": 3,
      "mostLikely": 5,
      "pessimistic": 9,
      "predecessors": ["A"]
    }
  ]
}
```

## 项目特点

本项目相较于基础任务调度程序，具有以下特点：

- 基于标准 PERT 三点估计进行工期分析
- 同时支持最早时间、最晚时间和时差分析
- 支持关键路径识别
- 支持多个起始任务和多个结束任务
- 支持最少人数计算与排班生成
- 支持可视化展示
- 支持数据保存与加载

## 注意事项

- 前置任务字段中填写的是任务 ID，不是任务名称
- 多个前置任务之间请使用英文逗号 `,`
- 输入数据必须满足 `O <= M <= P`
- 若依赖图存在循环，程序将无法计算
- 建议使用现代浏览器运行，以获得更好的显示效果
